Data Pipeline Visualizer

Click through the steps below to animate the lifecycle of a transaction—from user input to background synchronization and remote cloud confirm.

1. User Action

User logs a personal expense (e.g. ₹500 for groceries) or joins a shared bill splitting group.

2. Riverpod State

Local state notifier receives the dispatch and triggers optimistic UI updates instantly.

3. Hive local persistence

Transactions are written into Hive local boxes for instant load support and offline availability.

4. Sync queue

If database edits are made, an action (insert/update/delete) is appended to the local sync queue.

5. Supabase confirming

When online, the sync manager pops queue items, pushes to Supabase, and confirms local data state.

Relational Database Schema

Kanakku uses PostgreSQL tables configured inside Supabase with cascading relationships and row-level authorization security checks.

profiles Table

id UUID PK
display_name VARCHAR
avatar_url VARCHAR
currency VARCHAR (Default: INR)

groups Table

id UUID PK
name VARCHAR
invite_code VARCHAR (Unique 6-char)
created_by FK profiles.id

group_members Table

id UUID PK
group_id FK groups.id
user_id FK profiles.id
is_admin BOOLEAN

group_expenses Table

id UUID PK
group_id FK groups.id
amount DECIMAL
split_type equal | custom

Key Feature Highlights

The core modules driving Kanakku's premium user experience.

App Lock PIN Security

Secures personal transactions and settings data on startup. Supports changing credentials with dynamic verification flows.

Smart Financial Insights

Integrates monthly category data streams to evaluate runway months and categorizes spending profiles intelligently.

Group Splits Replication

Replicates split shared expenses automatically to personal transaction caches on confirmation to keep records in sync.

Multi-Currency Support

Syncs live exchange rates dynamically, presenting transactions in user preferred currency and showing base conversions.
